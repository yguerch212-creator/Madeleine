PARTIE 1 — SEO TECHNIQUE

1.1 Core Web Vitals

Les Core Web Vitals ont été mesurés via Google PageSpeed Insights (stratégie mobile). Résultat global : ÉCHEC — le site ne passe pas l'évaluation Core Web Vitals.

MétriqueValeur mesuréeSeuil « Bon »Verdict
LCP (Largest Contentful Paint)2,3 s≤ 2,5 s✅ Bon
INP (Interaction to Next Paint)333 ms≤ 200 ms⚠️ À améliorer
CLS (Cumulative Layout Shift)0,33≤ 0,1⚠️ À améliorer
FCP (First Contentful Paint)1,7 s≤ 1,8 s✅ Bon
TTFB (Time To First Byte)1,4 s≤ 0,8 s❌ Problème

Analyse :

  • LCP (2,3s) : Le plus gros élément visible (hero image / bannière promotionnelle) se charge dans les limites acceptables, mais reste proche du seuil. Comparé à des concurrents comme Quitoque (~1,8s), c'est en retrait.
  • INP (333ms) : L'interactivité est dégradée. Les clics (ajout au panier, navigation entre recettes) mettent trop de temps à produire un retour visuel. Cela est probablement dû au JavaScript lourd (React/Next.js, GTM, scripts tiers marketing).
  • CLS (0,33) : Les décalages de layout sont importants — 3× le seuil acceptable. Les images sans dimensions explicites, les polices custom (Agrandir Tight, Roboto) chargées en swap, et les bannières promotionnelles insérées dynamiquement provoquent ces sauts visuels.
  • TTFB (1,4s) : Le temps de réponse du serveur est élevé. Pour un site Next.js en SSR, cela suggère un CDN mal configuré ou un temps de calcul côté serveur trop long.

Recommandations :

  1. Réduire le JavaScript bloquant (code-splitting, lazy-loading des composants non-critiques)
  2. Ajouter des attributs width et height explicites sur toutes les images pour réduire le CLS
  3. Investiguer le TTFB élevé : cache CDN (Cloudflare/Fastly), optimisation SSR, pre-rendering des pages statiques
  4. Différer le chargement de GTM et des scripts marketing non-essentiels

1.2 Optimisation des images

⚠️ À améliorer — 29 images sur 76 sont trop lourdes (38% du total).

Constats :

  • HelloFresh utilise un CDN d'images dédié (img.hellofresh.com / media.hellofresh.com) avec transformation automatique via paramètres URL (w_256,q_100,f_auto,c_limit,fl_lossy).
  • Le format f_auto sert du WebP aux navigateurs compatibles — c'est une bonne pratique.
  • Cependant, certaines images de recettes et la hero image sont servies en haute qualité (q_100) sans nécessité, ce qui alourdit la page.
  • Les images des recettes (carrousel) ne sont pas toutes lazy-loadées.
  • Le logo est préchargé en 2 résolutions (96px et 256px via imageSrcSet) — bonne pratique.

Recommandations :

  1. Réduire la qualité à q_80 pour les images non-hero (gain estimé ~30% sur le poids)
  2. Implémenter le lazy-loading systématique (loading="lazy") sur toutes les images sous le fold
  3. Vérifier que les dimensions sont servies adaptées au viewport mobile (pas d'images desktop sur mobile)

1.6 Taux d'indexation

⚠️ Estimation partielle

  • Le sitemap référence 8 716 URLs.
  • La requête site:hellofresh.fr sur les moteurs de recherche retourne des résultats variés couvrant les principales sections (recettes, panier-repas, about, contact, FAQ, menu de la semaine).
  • Sans accès à Google Search Console, le taux exact d'indexation ne peut pas être déterminé.
  • Estimation : Étant donné la taille du site (8 700+ URLs) et le bon état technique (sitemaps, robots.txt), le taux d'indexation est probablement élevé (>80%).
  • Attention : les pages de recettes datant de 2018 dans le sitemap (ex : lastmod: 2018-05-16) pourraient avoir été désindexées si le contenu n'a pas été mis à jour.

1.7 Erreurs 404

⚠️ À améliorer — 4 erreurs 404 détectées.

  • 4 URLs retournent un code 404 lors du crawl.
  • C'est un nombre faible pour un site de 8 700+ pages, ce qui est positif.
  • Cependant, toute erreur 404 gaspille du budget de crawl et dégrade l'expérience utilisateur.

Recommandation : Identifier les 4 URLs en 404, mettre en place des redirections 301 vers les pages les plus proches ou la page d'accueil.

1.8 Sous-domaines

✅ Bon — Architecture cohérente

Sous-domaines identifiés :

  • www.hellofresh.fr — site principal (FR)
  • hubspot.hellofresh.com — formulaires de partenariat (HubSpot)
  • media.hellofresh.com / img.hellofresh.com — CDN images
  • cdn.hellofresh.com — assets statiques

L'architecture est propre : le contenu SEO est concentré sur www.hellofresh.fr, les assets sont sur des sous-domaines dédiés (pas de dilution d'autorité). Le blog n'est pas sur un sous-domaine séparé (il est intégré via /blog/ ou les stories), ce qui est une bonne pratique SEO.

PARTIE 2 — SEO ON-SITE (CONTENU)

2.1 Balise Title

✅ Bon

  • Title : « HelloFresh | La Box à Cuisiner, développée en France »
  • Longueur : 52 caractères (seuil recommandé : ≤ 65 caractères) ✅
  • Mots-clés : « HelloFresh » (marque) + « Box à Cuisiner » (mot-clé commercial) + « France » (localisation) — très bien ciblé.
  • Point fort : Le pipe (|) sépare la marque du descriptif, format classique et efficace.

Pages recettes : Le format « [Nom recette] Recette | HelloFresh » est cohérent et optimisé (ex : « Filet de poulet et pommes de terre au pesto Recette | HelloFresh »).

2.2 Meta Description

✅ Bon

  • Contenu : « Vos plats préférés livrés chez vous. Simplifiez vos repas avec nos recettes gourmandes. »
  • Longueur : 87 caractères (seuil recommandé : < 155 caractères) ✅
  • Incitatif : Ton direct, promesse claire (livraison + simplicité + gourmandise) — donne envie de cliquer.

Remarque : La description est peut-être un peu courte. On pourrait ajouter un CTA (« Découvrez nos 54+ recettes de la semaine ») pour occuper plus d'espace dans les SERPs et augmenter le CTR.

2.3 Structure des Headings

❌ Problème — Structure incohérente

BaliseNombre
H14
H25
H33
H4, H5, H60

Problèmes identifiés :

  • 4 balises H1 sur la page d'accueil — il devrait y en avoir une seule. Avoir plusieurs H1 dilue le signal SEO du titre principal.
  • Un H3 apparaît au-dessus de H1 dans le DOM, ce qui brise la hiérarchie sémantique.
  • Les H2 couvrent les sections principales (témoignages, FAQ, offre) mais sans cohérence de structure.

Recommandations :

  1. Garder un seul H1 par page (ex : « Des millions de repas livrés en France »)
  2. Transformer les H1 supplémentaires en H2
  3. Respecter la hiérarchie descendante (H1 → H2 → H3), jamais un H3 avant un H1

2.5 Attribut Alt des images

❌ Problème

  • 47,5% des images n'ont pas d'attribut alt — près de la moitié.
  • C'est un problème majeur pour l'accessibilité (WCAG) et le SEO image.
  • Les images de recettes sans alt manquent une opportunité de se positionner sur Google Images (« recette poulet pesto », etc.).

Recommandations :

  1. Ajouter un attribut alt descriptif sur chaque image de recette (ex : alt="Filet de poulet au pesto avec pommes de terre et salade")
  2. Pour les images décoratives, utiliser alt="" (alt vide, pas absent)
  3. Automatiser la génération des alt à partir du nom de la recette dans le CMS

2.6 Diversité HTML

✅ Bon

  • Le site utilise une variété d'éléments HTML : images, vidéos témoignages, carrousels, listes, icônes SVG, formulaires interactifs.
  • Pas de « pavé de texte » — le contenu est aéré avec des visuels bien placés.
  • Les pages de recettes contiennent des données structurées (ingrédients, calories, temps de préparation, allergènes, valeurs nutritionnelles).
  • Point d'amélioration : Pas de balisage Schema.org Recipe détecté en JSON-LD sur la page d'accueil. Les pages de recettes individuelles semblent utiliser des données structurées, mais cela mériterait une vérification via le Rich Results Test de Google.

Exemples d'URLs propres :

  • /recipes/recettes-faciles ✅
  • /panier-repas/menu-de-la-semaine ✅
  • /about/how-it-works ✅

Exemples d'URLs problématiques :

  • URLs de recettes avec des IDs hexadécimaux longs : /recipes/filet-de-poulet-et-pommes-de-terre-au-pesto-5afbf80830006c54c636b1d2 — l'ID MongoDB en fin d'URL n'apporte rien au SEO.

Recommandations :

  1. Supprimer les identifiants techniques des URLs de recettes
  2. Remplacer les underscores par des tirets
  3. Forcer les URLs en minuscules
  4. Translitérer les caractères accentués (é → e, ç → c)

2.8 Qualité du contenu rédactionnel

❌ Problème — Thin content

  • 108 URLs contiennent moins de 300 mots de contenu textuel.
  • La page d'accueil elle-même est relativement pauvre en texte (contenu principalement visuel et promotionnel).
  • Les pages de recettes individuelles compensent avec un contenu riche (ingrédients, étapes, valeurs nutritionnelles, allergènes) — c'est le point fort du site.

Points positifs :

  • Contenu evergreen : Les recettes sont du contenu froid et durable, pertinent toute l'année. ✅
  • Maillage interne : Bien orienté vers la conversion — les recettes renvoient vers les collections, les collections vers la commande. ✅
  • Pas de duplicate content apparent entre les recettes (chaque recette a un contenu unique).

Points négatifs :

  • Les pages « collection » (ex : /recipes/recettes-faciles) sont pauvres en contenu éditorial.
  • Absence de blog structuré avec des articles longs (guides, comparatifs, conseils nutritionnels) qui pourraient capter du trafic informationnel.

Recommandations :

  1. Enrichir les 108 pages thin content avec au minimum 300 mots de texte pertinent
  2. Créer une section blog/magazine avec des articles > 1000 mots (ex : « Comment manger équilibré en famille », « Guide des box repas en France »)
  3. Ajouter du contenu éditorial aux pages de collection de recettes

3.2 Évaluation des backlinks

✅ Bon — Profil naturel et diversifié

Qui cite hellofresh.fr :

  • Presse nationale : Capital.fr (article détaillé sur la stratégie de HelloFresh France), Le Figaro, 20 Minutes — liens éditoriaux de très haute qualité (DA 80+).
  • Sites de confiance : Trustpilot.com (36 355 avis, note 4/5), Wikipedia (page HelloFresh SE).
  • Blogs culinaires / lifestyle : Une-case-en-plus.com, Academy-ecommerce.com, etc. — typique des « mommy bloggers » et blogs food qui font des tests/avis. Liens souvent en dofollow via le programme d'affiliation.
  • Réseaux sociaux : Instagram (@hellofreshfrance, 178K abonnés), Facebook — signaux sociaux importants.
  • Partenaires institutionnels : FFBB (Fédération Française de Basket-Ball) — HelloFresh est fournisseur officiel des équipes de France, ce qui génère des backlinks sportifs de qualité.
  • Portails emploi : Welcome to the Jungle (page entreprise), Indeed, Glassdoor.

Types d'ancres :

  • Ancres de marque : « HelloFresh », « HelloFresh France » — majoritaires (sain)
  • Ancres URL : « hellofresh.fr » — fréquent
  • Ancres descriptives : « box repas », « livraison de repas », « code promo HelloFresh » — présentes via les blogs affiliés
  • Ancres génériques : « cliquez ici », « site officiel » — minoritaires

Ratio dofollow / nofollow : La majorité des liens de presse et de blogs sont en dofollow. Les liens Trustpilot et réseaux sociaux sont en nofollow, ce qui est normal et sain pour un profil naturel.

Point de vigilance : L'analyse SingleGrain (2021) sur 1 million de backlinks HelloFresh (toutes versions confondues) notait un cluster de Trust Flow à 27, signe de liens site-wide potentiellement dupliqués. Il convient de surveiller les backlinks automatiques (scrapers, annuaires de mauvaise qualité).

3.3 Opportunités de netlinking

⚠️ Axes d'amélioration identifiés

1. Link Baiting (contenu qui attire naturellement des liens) :

  • Infographies : Créer des infographies sur la nutrition, le gaspillage alimentaire, les habitudes culinaires des Français — contenu très partagé et cité.
  • Études propriétaires : HelloFresh dispose de données uniques (recettes les plus populaires par région, tendances alimentaires). Publier une « Étude HelloFresh sur les goûts culinaires des Français » générerait des reprises presse.
  • Outils gratuits : Un calculateur de budget courses vs HelloFresh, un planificateur de repas de la semaine — outils linkables.

2. Guest Blogging :

  • Contribuer à des blogs culinaires reconnus (Marmiton, 750g, Cuisine Actuelle) avec des articles signés par les chefs HelloFresh.
  • Partenariats avec des blogs nutrition/santé (Yuka, Healthline FR) pour des articles sur l'alimentation équilibrée.
  • Articles invités sur des sites lifestyle (Madame Figaro, Elle à Table) pendant les temps forts (rentrée, fêtes).

3. Échanges et partenariats :

  • Programme d'affiliation existant : HelloFresh a déjà un programme affilié (/about/affiliates) — c'est un levier de backlinks naturels. Le programme pourrait être élargi aux micro-influenceurs (1K-10K abonnés) pour diversifier les sources.
  • Partenariats sportifs : Étendre le partenariat FFBB à d'autres fédérations (athlétisme, rugby) pour des backlinks institutionnels.
  • Partenariats RSE : Collaborations avec des associations anti-gaspillage (Too Good To Go, Emmaüs) — backlinks à fort Trust Flow + image de marque.
  • Relations presse digitales : Communiqués réguliers aux rédactions food/tech/business pour maintenir la couverture médiatique et le flux de backlinks éditoriaux.

4.2 Top 5 des actions prioritaires

  1. Corriger la structure des headings : Un seul H1 par page, hiérarchie descendante stricte. Impact direct sur l'interprétation sémantique par Google. Effort : faible. Impact : élevé.
  2. Ajouter les attributs alt manquants : 47,5% des images sans alt = quasi la moitié du potentiel SEO image perdu + problème d'accessibilité. Automatisable via le CMS. Effort : moyen. Impact : élevé.
  3. Améliorer les Core Web Vitals (INP + CLS) : Réduire le JavaScript bloquant pour l'INP, ajouter des dimensions aux images et stabiliser les bannières dynamiques pour le CLS. Effort : élevé. Impact : élevé (facteur de ranking Google).
  4. Enrichir le contenu thin : Les 108 pages à moins de 300 mots doivent être enrichies ou consolidées. Créer une stratégie de contenu blog/magazine pour capter le trafic informationnel. Effort : élevé. Impact : moyen-élevé.
  5. Nettoyer les URLs : Supprimer les IDs techniques, forcer les minuscules, remplacer les underscores. Mettre en place des redirections 301 pour les anciennes URLs. Effort : moyen. Impact : moyen.

4.3 Conclusion

HelloFresh France bénéficie d'une base technique solide (HTTPS, sitemap, mobile responsive, CDN images) et d'un profil de backlinks enviable pour le marché français (presse nationale, partenariats sportifs, programme d'affiliation actif).

Les principaux axes d'amélioration sont on-site : la structure sémantique (headings), l'accessibilité des images (attributs alt), la performance JavaScript (Core Web Vitals), et la stratégie de contenu (thin content, absence de blog structuré).

En corrigeant ces points, HelloFresh France pourrait significativement améliorer son positionnement organique, notamment sur les requêtes informationnelles (recettes, nutrition, alimentation équilibrée) qui représentent un volume de recherche considérable en France.
